# 3️⃣ Thread vs Task (Very Important Interview Question)

| Thread          | Task               |
| --------------- | ------------------ |
| OS managed      | .NET managed       |
| Heavy           | Lightweight        |
| Blocking        | Async non-blocking |
| Low scalability | High scalability   |

```csharp
new Thread(Work).Start();

Task.Run(Work);
```

✅ **Always prefer Task over Thread**

# 4️⃣ Thread Pool Internals

.NET maintains **global thread pool**.

* Worker threads → CPU work
* IO threads → async IO

```csharp
ThreadPool.GetAvailableThreads(out int worker, out int io);
```

### Thread Pool Algorithm

Uses **work stealing** for load balancing.

# 6️⃣ Async / Await Internals (Very Important)

### What happens internally?

Compiler:

* Converts method → **State Machine**
* Frees thread on `await`
* Resumes when awaited task completes

```csharp
await Task.Delay(1000);
```

Thread is **not blocked** → returned to pool → **scalability boost** 🚀

---

# 7️⃣ Synchronization Context

Controls **where continuation runs after await**.

| Environment  | Context   |
| ------------ | --------- |
| ASP.NET Core | ❌ None    |
| Desktop UI   | UI Thread |
| Console      | ❌ None    |

```csharp
await Task.Delay(1000).ConfigureAwait(false);
```

Avoids **deadlocks**.

# 13️⃣ Producer Consumer Pattern (Real Interview Favorite)

```csharp
BlockingCollection<int> queue = new(100);

Task.Run(() =>
{
    for (int i = 0; i < 100; i++)
        queue.Add(i);
    queue.CompleteAdding();
});

Task.Run(() =>
{
    foreach (var item in queue.GetConsumingEnumerable())
        Console.WriteLine(item);
});
```

# 14️⃣ Background Worker (ASP.NET Core Hosted Service)

```csharp
public class Worker : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken token)
    {
        while (!token.IsCancellationRequested)
        {
            await Task.Delay(5000, token);
            Console.WriteLine("Worker running...");
        }
    }
}
```

## 🧠 Problem 1: Print 1 to 100 using 4 threads sequentially

```csharp
int number = 1;
object _lock = new();

void Print()
{
    while (true)
    {
        lock (_lock)
        {
            if (number > 100) return;
            Console.WriteLine(number++);
        }
    }
}

Parallel.Invoke(Print, Print, Print, Print);
```

# 🔥 MOST IMPORTANT INTERVIEW QUESTIONS

### Q1. Why async/await improves scalability?

Because threads are **not blocked**, they return to pool → more requests served → high throughput.

---

### Q2. Difference: Task.Run vs async method?

| Task.Run       | async         |
| -------------- | ------------- |
| CPU-bound      | IO-bound      |
| Creates thread | No new thread |
| Parallel work  | Non-blocking  |

---

### Q3. When NOT to use multithreading?

* DB operations
* HTTP calls
* File I/O

Use **async/await instead**

---

### Q4. How deadlocks occur in async code?

Using `.Result` or `.Wait()`

```csharp
var data = GetDataAsync().Result; ❌ DEADLOCK
```

Fix:

```csharp
await GetDataAsync(); ✅
```
