```

   ↓
Kestrel Server
   ↓
Middleware Pipeline
   ↓
Routing
   ↓
Controller / Endpoint

Response
```

**Key Components:**

* Kestrel (Web Server)
* Middleware
* Routing Engine
* MVC Pipeline
* Dependency Injection
* Logging & Configuration

# 2️⃣ Middleware Pipeline Deep Dive

### What happens internally?

Each middleware:

* Receives HttpContext
* Calls next middleware
* Can short-circuit pipeline

```csharp
public async Task Invoke(HttpContext context)
{
    // Pre-processing
    await _next(context);
    // Post-processing
}
```

**Pipeline Example:**

```csharp
app.UseAuthentication();
app.UseAuthorization();
app.UseEndpoints();
```

⚠ Order matters!

# 5️⃣ Async / Await Internals

**What happens internally?**

* State machine generated by compiler
* Non-blocking thread
* Thread returned to pool while awaiting


Thread not blocked → **scalability boost**

# 32️⃣ Background Worker (Hosted Service)

```csharp

{
    protected override async Task ExecuteAsync(CancellationToken token)
    {

        {
            await Task.Delay(5000);
        }
    }
}
```
